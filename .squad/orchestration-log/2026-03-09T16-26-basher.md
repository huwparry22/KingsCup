## Testing Patterns Established

3. **Edge Cases:** Explicitly test boundaries (empty deck, invalid states, max players)
4. **Probabilistic Tests:** Shuffle uses probabilistic validation to avoid flaky tests (1 in 132,600 chance of false positive)
5. **Integration Tests:** Game flow tests validate state machine transitions (not isolated unit tests)
