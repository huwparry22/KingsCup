## Key Decisions

1. **Blazor Web App (not WASM)** with InteractiveServer for real-time multiplayer via SignalR
2. **Domain-driven design** with GameRoom as aggregate root
3. **In-memory storage** using ConcurrentDictionary (ephemeral sessions, no database)
4. **6-character room codes** (alphanumeric, case-insensitive, excludes I/O/0/1)
5. **Server-authoritative state** to prevent client-side cheating
